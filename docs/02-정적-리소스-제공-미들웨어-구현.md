## 2단계: 정적 리소스 제공 미들웨어 직접 구현하기

학습 목표
- 미들웨어의 위치와 역할을 이해한다. (미들웨어는 요청이 컨트롤러에 도달하기 전에 '문지기'처럼 검사하고 처리하는 중간 단계예요.)
- 정적 파일 서빙 과정을 코드로 직접 구현/확장한다.
- MIME 타입, 응답 헤더, 에러 처리의 중요 포인트를 익힌다.

핵심 코드 위치
- `src/resource-response/resource-response.middleware.ts`
- `src/app.module.ts` (미들웨어 등록)

요약: 동작 흐름
1) 요청 메소드 확인 (GET만 처리)
2) 원본 URL에서 파일 경로 도출 (`const resourcePath = .${url}`)
3) 파일 존재 여부 검사 → 존재하면 확장자 추출·MIME 매핑 → 적절한 `Content-Type` 설정
4) 바이트 응답 전송 또는 404 JSON 반환

코드 주요 포인트
- 로깅: `Logger(ResourceResponseMiddleware.name)` 인스턴스로 디버그 로그 출력 (로깅은 앱이 무슨 일을 하는지 '일기장'처럼 기록해 문제 해결에 도움을 줍니다.)
- 메소드 필터: `if (method !== 'GET') return next()`
- 경로/확장자 처리: `url.substring(url.lastIndexOf('.') + 1).toLowerCase()`
- MIME 매핑: `mimeTypes` 상수에서 `Content-Type`을 찾아 헤더 설정
- 실패 처리: 리소스 미존재 시 JSON 바디와 404 상태코드 반환

실습 1: MIME 타입 확장하기
1) 다음 타입을 추가해보자 (MIME 타입은 파일 종류를 알려주는 '라벨'로, 브라우저가 어떻게 해석할지 결정해줍니다.)
   - `woff: 'font/woff'`
   - `woff2: 'font/woff2'`
   - `map: 'application/json'` (소스맵)
2) CSS에서 웹폰트를 참조하거나 소스맵 요청을 보내 테스트한다.

실습 2: 캐시 헤더 추가하기
- 정적 파일은 브라우저 캐시를 활용하면 성능이 좋아진다.
- 예시: 1시간 캐시
```ts
res.setHeader('Cache-Control', 'public, max-age=3600, immutable');
```
- `curl -I http://localhost:3000/assets/css/index.css`로 헤더를 확인한다.

실습 3: 보안 — 경로 정규화와 베이스 디렉터리 제한
- 잠재적 위험: `req.originalUrl`을 그대로 파일 경로로 이어붙일 경우, 경로 역참조(`..`) 시도가 있을 수 있다. (이건 해커가 서버의 다른 파일을 보려는 시도예요. 보안 코드는 이런 걸 막아줍니다.)
- 개선 아이디어
  - `path.normalize()`로 경로 정규화
  - `assets` 디렉터리 밖으로 나가지 못하도록 베이스 디렉터리를 강제
```ts
import * as path from 'node:path';

const baseDir = path.resolve(process.cwd(), 'assets');
const requestPath = path.normalize(url).replace(/^\\+|^\/+/, '');
const absolutePath = path.join(baseDir, requestPath);
if (!absolutePath.startsWith(baseDir)) {
  // 베이스 디렉터리 밖 접근 시도 → 404 처리
}
```

실습 4: 스트리밍으로 개선해보기(선택)
- 현재는 `fs.readFileSync`로 바이트를 메모리로 읽은 뒤 응답한다.
- 큰 파일의 경우 `createReadStream()`으로 스트리밍 전송을 검토해보자.
```ts
import * as fs from 'node:fs';

res.setHeader('Content-Type', contentType);
const stream = fs.createReadStream(absolutePath);
stream.pipe(res);
```

실습 5: 디렉터리 인덱스 처리(선택)
- URL이 `/assets/` 또는 `/assets/some/`처럼 슬래시로 끝나는 경우, 해당 경로의 `index.html`을 찾아 응답하도록 확장한다.
- 구현 포인트: 슬래시로 끝나면 `path.join(requestPath, 'index.html')` 검사

추가 이해: 왜 미들웨어로 했을까?
- 컨트롤러에서 `express.static` 비슷한 기능을 직접 구현할 수도 있지만, 정적 파일 서빙은 컨트롤러 이전 단계에서 빠르게 단락시키는 것이 더 효율적이다. (미들웨어로 하면 앱의 '본론'인 동적 처리 전에 간단한 일을 끝낼 수 있어요.)
- 이 프로젝트는 “웹서버의 원리”를 학습하려고 직접 구현하여 가시성을 높였다.
