## 3단계: 컨트롤러/유틸/테스트로 응답 흐름 완성하기

학습 목표
- 컨트롤러가 문자열(HTML) 응답을 반환하는 방식을 이해한다. (컨트롤러는 요청을 받아 '무엇을 할지' 결정하는 앱의 '뇌'예요.)
- 파일 유틸 함수의 역할과 동기 IO의 특성을 이해한다.
- E2E 테스트로 정적 리소스/오류 케이스를 검증한다.

핵심 코드 위치
- 컨트롤러: `src/app.controller.ts`
- 유틸: `src/utils/file-util.utils.ts`
- (테스트) `test/app.e2e-spec.ts` — 현재 비어 있으므로 실습에서 작성

컨트롤러 살펴보기
- 루트 경로(`/`) 핸들러는 `readFileString('./assets/index.html')`로 파일을 읽어 문자열로 반환한다.
- 장점: 라우트가 단순하고 학습에 적합
- 제약: 대용량 파일에 비효율적이며 템플릿 렌더링/헤더 제어에 한계

유틸 함수 정리
- `readFileString(path: string): string` → 텍스트 파일을 UTF-8로 동기 읽기 (동기 읽기는 '기다리며' 파일을 가져오지만, 서버가 바쁠 때는 지연을 일으킬 수 있어요. 비동기는 '기다리지 않고' 다른 일을 하며 결과를 받습니다.)
- `readFileBytes(path: string): Buffer` → 바이트 배열로 동기 읽기
- `exists(path: string): boolean` → 파일 존재 여부
- 동기 IO는 호출 시점에 이벤트 루프를 블로킹하므로, 학습 목적을 넘어 대용량/고부하 서비스에서는 비동기/스트리밍으로 전환 필요

실습 1: 루트 페이지 내용 테스트 작성
1) 테스트 의존성은 이미 `package.json`에 구성되어 있다. 다음 스펙을 작성한다. (E2E 테스트는 실제 앱을 켜놓고 요청을 보내 동작을 확인하는 '모의 사용자' 테스트예요.)
```ts
// test/app.e2e-spec.ts
import { Test, TestingModule } from '@nestjs/testing';
import { INestApplication } from '@nestjs/common';
import * as request from 'supertest';
import { AppModule } from '../src/app.module';

describe('App (e2e)', () => {
  let app: INestApplication;

  beforeAll(async () => {
    const moduleFixture: TestingModule = await Test.createTestingModule({
      imports: [AppModule],
    }).compile();

    app = moduleFixture.createNestApplication();
    await app.init();
  });

  afterAll(async () => {
    await app.close();
  });

  it('GET / should include title text', async () => {
    const res = await request(app.getHttpServer()).get('/');
    expect(res.status).toBe(200);
    expect(res.text).toContain('라퓨타의 페이지');
  });
});
```
2) 실행
```powershell
npm run test:e2e
```

실습 2: 정적 리소스와 404 케이스 테스트 추가
```ts
it('GET /assets/css/style.css should be CSS', async () => {
  const res = await request(app.getHttpServer()).get('/assets/css/index.css');
  expect(res.status).toBe(200);
  expect(res.header['content-type']).toMatch(/text\/css/);
  expect(res.text).toContain('.container');
});

it('GET /assets/unknown.js should return 404 JSON', async () => {
  const res = await request(app.getHttpServer()).get('/assets/unknown.js');
  expect(res.status).toBe(404);
  expect(res.header['content-type']).toMatch(/application\/json/);
  expect(res.body).toMatchObject({ statusCode: 404 });
});
```

실습 3: 간단한 API 엔드포인트 추가(선택)
1) 헬스체크 추가
```ts
// src/app.controller.ts (예시)
import { Controller, Get } from '@nestjs/common';

@Controller()
export class AppController {
  @Get('/api/health')
  health() {
    return { status: 'ok' };
  }
}
```
2) 테스트 작성
```ts
it('GET /api/health returns ok', async () => {
  const res = await request(app.getHttpServer()).get('/api/health');
  expect(res.status).toBe(200);
  expect(res.body).toEqual({ status: 'ok' });
});
```

실습 4: 헤더/인코딩 점검(선택)
- 루트 핸들러가 문자열을 반환하므로, HTML로 렌더되어야 한다.
- 필요 시 컨트롤러에 `@Header('Content-Type', 'text/html; charset=utf-8')`를 지정하는 방식도 검토한다.

운영 관점 팁
- 정적 파일은 CDN 또는 `express.static` 같은 검증된 미들웨어로 처리하고, 애플리케이션은 동적 API에 집중시키는 것이 일반적이다. (CDN은 파일을 빠르게 전 세계에 배포해주는 '클라우드 저장소'처럼 작동해 서버 부하를 줄여줍니다.)
- 이 프로젝트는 원리를 학습하기 위해 직접 구현했으며, 테스트를 통해 동작을 안전하게 확장하는 방법을 익힌다.
